# SuperMarket 超市智慧运营管理系统 — 软件设计文档

## 0. 需求分析

### 0.1 业务需求概述

随着零售行业数字化转型的加速，传统超市在日常运营中面临诸多痛点：

- **信息孤岛**：商品、库存、采购、销售等数据分散在不同系统或纸质记录中，无法形成统一的经营视图
- **人工差错频发**：手工记账容易出现数量错误、价格偏差、重复录入等问题，尤其在盘点和对账环节
- **决策滞后**：缺乏实时数据支撑，管理者无法及时了解销售趋势、库存预警、员工绩效等关键指标
- **协作效率低**：采购员、理货员、收银员之间缺乏协同平台，信息传递依赖口头或纸质单据
- **供应链不透明**：供应商管理混乱，采购审批流程冗长，库存补货缺乏数据依据

本系统旨在构建一套覆盖超市核心业务全流程的智慧运营管理平台，通过信息化手段实现商品管理、供应商管理、采购管理、库存管理、销售管理五大核心业务的数字化闭环，并引入 AI 智能助手和数据可视化能力，全面提升运营效率和决策质量。

## 1. 系统概述

### 1.1 项目背景

随着零售行业数字化转型的加速，传统超市在商品管理、库存监控、采购决策和销售分析等环节面临效率低下、信息孤岛、人工差错频发等问题。本系统旨在构建一套覆盖超市核心业务全流程的智慧运营管理平台，通过信息化手段提升运营效率、降低管理成本、辅助经营决策。

### 1.2 系统目标

- **全流程覆盖**：涵盖商品管理、供应商管理、采购管理、库存管理、销售管理五大核心业务模块
- **多角色协作**：支持店长、采购员、理货员、收银员四种角色，各司其职、权限隔离
- **数据驱动决策**：通过仪表盘、销售趋势、库存预警、智能补货建议等数据可视化手段辅助经营
- **AI 智能助手**：集成大语言模型，支持自然语言查询系统数据、智能创建商品/供应商等写操作
- **通知消息系统**：支持系统通知、库存预警、采购状态等多类型消息推送与已读管理

### 1.3 用户角色定义

| 角色 | 标识 | 职责说明 | 可访问模块 |
|------|------|----------|------------|
| 店长 | `admin` | 系统最高权限，负责审批采购单、管理用户、配置系统、查看全局报表 | 全部模块 |

### 2.3 数据流模式

系统采用 React Router v7 的 **loader/action** 模式实现服务端渲染（SSR）：

```
用户操作 → <Form> / useFetcher → action 处理 → redirect / 返回数据
页面加载 → loader 读取数据 → SSR 渲染 → 客户端 hydrate
```

- **loader**：在服务端执行，负责数据查询和权限校验，返回的数据自动注入组件的 `loaderData` props
- **action**：处理表单提交（POST），执行写操作后通过 `throw redirect()` 或 `throw flashRedirect()` 跳转
- **useFetcher**：用于不触发页面跳转的 AJAX 操作（如删除、状态切换），支持乐观更新

### 2.4 认证方案

采用基于 Cookie 的 Session 认证：

1. 用户提交用户名/密码到 `/login`
2. 服务端通过 `bcrypt.compare` 验证密码
3. 验证通过后将 `userId` 写入 httpOnly Cookie（`__session`）
4. 后续请求通过 `getUserSession()` 从 Cookie 读取 userId，查询数据库获取用户信息
5. `requireRole(request, roles)` 在 loader/action 中强制执行角色权限校验
6. Session 有效期 24 小时，`SameSite=Lax`，生产环境启用 `Secure` 标志

### 2.6 服务端架构详解

#### 请求处理流程

```
HTTP Request
    ↓
React Router 路由匹配（文件名 → URL 路径）
    ↓
┌─────────────────────────────────────┐
│ loader（GET 请求）                    │
│  1. getUserSession(request)          │  ← 从 Cookie 读取 userId
│  2. requireUser(request)             │  ← 未登录则 redirect("/login")
│  3. requireRole(request, roles)      │  ← 权限不足则 Response 403
│  4. db.model.findMany/aggregate      │  ← Prisma 查询数据库
│  5. 数据序列化（Decimal→Number 等）    │
│  6. return { user, data }            │  ← 注入 loaderData
└─────────────────────────────────────┘
    ↓
SSR 渲染 → HTML → 客户端 hydrate
    ↓
┌─────────────────────────────────────┐
│ action（POST 请求）                   │
│  1. requireRole(request, roles)      │
│  2. request.formData()               │  ← 解析表单数据
│  3. intent 字段路由到具体操作          │
│  4. 数据库写操作（create/update/delete）│
│  5. throw redirect() 或 return {}    │
└─────────────────────────────────────┘
```

#### 数据序列化约定

数据库查询结果在返回前端前需要序列化处理：

| 数据类型 | 序列化方式 | 示例 |
|---------|-----------|------|
| Decimal | `Number(value)` | 商品价格 `Number(p.price)` |
| Bytes（头像） | `hasAvatar: boolean` | 避免传输二进制数据，通过 `/api/avatar` 单独获取 |
| DateTime | `.toISOString()` | 创建时间、更新时间 |
| null | 保持不变 | 可选字段（description、remark 等） |

## 4. 模块设计

### 4.1 认证模块　← 负责人：李衡

**核心文件**：`app/lib/auth.ts`、`app/lib/auth.server.ts`、`app/lib/session.server.ts`

**功能**：
- 用户登录（用户名/密码验证，bcrypt 哈希比对）
- Session 管理（Cookie 存储 userId，24 小时有效期）
- 权限守卫函数：`requireUser()` 确保登录，`requireRole(roles)` 确保角色权限
- 登出（销毁 Session，重定向到登录页）
- 登录页装饰卡片浮动动画（开发中）：CSS 浮动动画（不同周期 5-8s，不同延迟 0-1.5s），使用 `will-change-transform` GPU 加速
- 登录页粒子背景（开发中）：Canvas 2D 粒子系统（30/60 个粒子，连线效果）

**类型定义**：
```typescript
type AuthUser = Pick<User, "id" | "username" | "name" | "role"> & {
  hasAvatar: boolean;  // 是否有头像（避免传输二进制数据）
};
```

**页面流程**：
```
未登录用户 → 访问任意页面 → requireUser() 检测无 Session
→ redirect("/login") → 用户输入用户名/密码
→ action 验证 bcrypt.compare → 成功：写入 Cookie → redirect("/dashboard")
→ 失败：返回 { error } → 前端 toast 提示
```

**核心代码（李衡）**：

```typescript
// auth.server.ts — 权限守卫
export async function requireUser(request: Request): Promise<AuthUser> {
  const session = await getUserSession(request);
  if (!session?.userId) throw redirect("/login");
  const user = await db.user.findUnique({
    where: { id: session.userId },
    select: { id: true, username: true, name: true, role: true, avatar: true },
  });
  if (!user) throw redirect("/login");
  return { ...user, hasAvatar: !!user.avatar };
}

export async function requireRole(request: Request, roles: Role[]): Promise<AuthUser> {
  const user = await requireUser(request);
  if (!roles.includes(user.role)) throw new Response("权限不足", { status: 403 });
  return user;
}

// login.tsx — 登录 action
export async function action({ request }: Route.ActionArgs) {
  const formData = await request.formData();
  const username = formData.get("username") as string;
  const password = formData.get("password") as string;

  const user = await db.user.findUnique({ where: { username } });
  if (!user) return { error: "用户名或密码错误" };

  const valid = await bcrypt.compare(password, user.password);
  if (!valid) return { error: "用户名或密码错误" };

  const { createUserSession } = await import("~/lib/session.server");
  return createUserSession(user.id, "/dashboard");
}
```

// sales.new.tsx — 结账（事务：创建订单 + 明细 + 扣减库存 + 记录日志）
export async function action({ request }: Route.ActionArgs) {
  const user = await requireRole(request, ["admin", "cashier"]);
  const formData = await request.formData();
  const itemsJson = formData.get("items") as string;
  const paymentMethod = formData.get("paymentMethod") as string || "cash";
  const items = JSON.parse(itemsJson) as Array<{ productId: number; quantity: number }>;

  if (!items || items.length === 0) return { error: "购物车不能为空" };

  // 查询商品价格和库存
  const products = await db.product.findMany({
    where: { id: { in: items.map(i => i.productId) } },
    include: { inventory: true },
  });

  // 逐项校验库存
  for (const item of items) {
    const p = products.find(p => p.id === item.productId);
    if (!p) return { error: `商品不存在` };
    const stock = p.inventory?.quantity ?? 0;
    if (stock < item.quantity) return { error: `${p.name} 库存不足：当前 ${stock}${p.unit}，需要 ${item.quantity}` };
  }

  const totalAmount = items.reduce((sum, item) => {
    const p = products.find(p => p.id === item.productId)!;
    return sum + item.quantity * Number(p.price);
  }, 0);

  await db.$transaction(async (tx) => {
    const order = await tx.saleOrder.create({
      data: { userId: user.id, totalAmount, paymentMethod, paymentStatus: "paid" },
    });
    await tx.saleOrderItem.createMany({
      data: items.map(item => {
        const p = products.find(p => p.id === item.productId)!;
        return { saleOrderId: order.id, productId: item.productId,
                 quantity: item.quantity, unitPrice: p.price };
      }),
    });
    // 扣减库存 + 记录出库日志
    for (const item of items) {
      await tx.inventory.update({
        where: { productId: item.productId },
        data: { quantity: { decrement: item.quantity } },
      });
      await tx.inventoryLog.create({
        data: { productId: item.productId, type: "OUT", quantity: item.quantity,
                reason: `销售出库 SO-${String(order.id).padStart(4, "0")}`, userId: user.id },
      });
    }
  });

  return flashRedirect("/sales", { type: "success", message: "收款成功" });
}

// app/lib/ai-assistant.server.ts — callLLM()
async function callLLM(config: AIConfig, messages: ChatMessage[]) {
  const decryptedKey = decrypt(config.apiKey);
  const baseUrl = config.baseUrl || providerBaseUrls[config.provider] || "";
  const protocol = config.protocol || "openai";

  const headers: Record<string, string> = { "Content-Type": "application/json" };
  const body: Record<string, unknown> = { model: config.model, messages, max_tokens: 2048 };

  if (protocol === "anthropic") {
    headers["x-api-key"] = decryptedKey;
    headers["anthropic-version"] = "2023-06-01";
    // Anthropic 协议：system 独立于 messages
    const systemMsg = messages.find((m) => m.role === "system");
    body.system = systemMsg?.content || "";
    body.messages = messages.filter((m) => m.role !== "system");
  } else {
    headers["Authorization"] = `Bearer ${decryptedKey}`;
    // OpenAI 协议：tools 参数定义可调用函数
    body.tools = tools;
    body.tool_choice = "auto";
  }

  const res = await fetch(baseUrl, { method: "POST", headers, body: JSON.stringify(body) });
  const data = await res.json();

  if (protocol === "anthropic") {
    return { content: data.content?.[0]?.text || "", toolCalls: data.content?.filter((c: any) => c.type === "tool_use") };
  }
  return {
    content: data.choices?.[0]?.message?.content || "",
    toolCalls: data.choices?.[0]?.message?.tool_calls,
  };
}

**已读管理**：
- 使用 NotificationRead 表记录已读状态

  const user = await getUserSession(request);
  const now = new Date();
  const today = new Date(now.getFullYear(), now.getMonth(), now.getDate());

  // 3 个并行查询：动态通知数据
  const [lowStockCount, pendingPurchaseCount, todaySalesCount, readRecords] = await Promise.all([
    db.inventory.count({ where: { quantity: { lt: 10 } } }),
    db.purchaseOrder.count({ where: { status: "pending" } }),
    db.saleOrder.count({ where: { createdAt: { gte: today } } }),
    db.notificationRead.findMany({ where: { userId: user.id }, select: { notificationId: true } }),
  ]);

  const notifications: Notification[] = [];
  if (lowStockCount > 0) notifications.push({ id: "stock-low", type: "stock", title: "库存预警", ... });
  if (pendingPurchaseCount > 0) notifications.push({ id: "purchase-pending", type: "purchase", title: "待审批采购单", ... });

  // 合并持久化通知（管理员创建的）
  const persistent = await db.notification.findMany({
    where: { OR: [{ targetRole: null }, { targetRole: user.role }] },
    orderBy: { createdAt: "desc" }, take: 20,
  });
  for (const n of persistent) {
    notifications.push({ id: `notif-${n.id}`, type: n.type, title: n.title, description: n.content, ... });
  }

  return Response.json({ notifications });
}

## 7. 安全设计

### 7.1 密码加密

- 使用 bcryptjs 进行密码哈希
- 默认 salt rounds: 10
- 登录时通过 `bcrypt.compare(plainText, hashedPassword)` 比对

### 7.2 API Key 加密存储

- 算法：AES-256-GCM（128 位认证标签）
- IV：每次加密随机生成 12 字节
- 格式：`iv:tag:ciphertext`（全部 hex 编码，以 `:` 分隔）
- 密钥来源：环境变量 `ENCRYPTION_KEY`（必须为 64 位十六进制字符，即 32 字节）

### 7.3 Session 安全

- Cookie 属性：`httpOnly`、`SameSite=Lax`
- 生产环境启用 `Secure` 标志
- 有效期：24 小时
- 存储内容：仅 `userId`（不存储敏感信息）

### 7.4 角色权限控制

- **路由级**：每个 loader/action 通过 `requireRole(request, roles)` 校验
- **组件级**：侧边栏导航通过 `item.roles.includes(user.role)` 过滤
- **工具级**：AI 写操作工具通过 `requiredRoles` 元数据校验，执行前检查用户角色
- **数据级**：查询条件中过滤用户可见范围（如采购员只能看到自己创建的采购单）

## 11. 性能优化

### 11.1 服务端优化

| 优化策略 | 实现方式 | 效果 |
|---------|---------|------|
| 查询字段精确控制 | `select` 仅返回需要的字段 | 减少数据传输量（如 users.tsx 避免查询 avatar Bytes） |
| 分页查询 | 所有列表页统一 `PAGE_SIZE = 20` | 避免一次加载过多数据 |
| 并行查询 | `Promise.all` 批量执行独立查询 | dashboard.tsx 中 9 个查询并行，减少总响应时间 |
| 事务保证一致性 | `$transaction` 包裹多步写操作 | 采购入库、销售结账的原子性 |
| 聚合查询 | `count`、`aggregate`、`groupBy` | 仪表盘统计使用数据库聚合而非应用层计算 |

### 11.2 前端优化

| 优化策略 | 实现方式 | 效果 |
|---------|---------|------|
| 组件懒加载 | `lazy(() => import("../charts"))` + `Suspense` | Charts 组件按需加载，减少首屏 JS 体积 |
| 骨架屏 | `PageSkeleton`、`TableSkeleton` | 减少用户感知的加载等待时间 |
| GPU 加速 | `will-change-transform` | 登录页粒子动画和浮动卡片使用 GPU 渲染 |
| 图片压缩 | Canvas 压缩至 256×256 JPEG 0.8 | 头像上传前压缩，减少传输体积 |
| 按钮反馈 | `active:scale-[0.98]` | 即时的视觉点击反馈 |

### 11.3 SSR 优化

- React Router v7 SSR 模式：服务端渲染完整 HTML，客户端 hydrate 添加交互
- loader 在服务端执行，减少客户端 API 调用次数
- 首屏渲染无客户端 JavaScript 阻塞

## 13. 错误处理与异常设计

### 13.1 认证异常

| 异常场景 | 处理方式 | 用户感知 |
|---------|---------|---------|
| 未登录访问受保护页面 | `requireUser()` → `throw redirect("/login")` | 自动跳转到登录页 |
| 权限不足 | `requireRole()` → `throw new Response("权限不足", { status: 403 })` | 显示 403 错误页面 |
| Session 过期（24 小时） | Cookie 自动过期 → 下次请求触发未登录逻辑 | 下次操作时跳转登录页 |

### 13.2 表单验证异常

```typescript
// action 中的验证模式
export async function action({ request }: Route.ActionArgs) {
  const formData = await request.formData();
  const name = formData.get("name") as string;

  // 验证失败：返回错误对象
  if (!name || name.trim() === "") {
    return { error: "名称不能为空", intent: "create" };
  }

  // 唯一约束冲突
  try {
    await db.category.create({ data: { name } });
  } catch (e) {
    if (e.code === "P2002") {
      return { error: "分类名称已存在", intent: "create" };
    }
  }

  return { ok: true, intent: "create" };
}
```

前端通过 `useFetcherToast` Hook 自动处理：
```typescript
// use-fetcher-toast.ts
useEffect(() => {
  if (fetcher.data?.error) toast.error(fetcher.data.error);
  if (fetcher.data?.ok) toast.success("操作成功");
}, [fetcher.data]);
```

用于操作完成后跳转页面时携带一次性提示消息：

### 13.4 数据库异常

| 异常类型 | Prisma 错误码 | 处理方式 |
|---------|--------------|---------|
| 唯一约束冲突 | P2002 | 返回用户友好的重复提示（如"分类名称已存在"） |
| 外键约束失败 | P2003 | 返回关联数据提示（如"该分类下还有商品，无法删除"） |
| 记录不存在 | P2025 | 返回 404 或"数据不存在"提示 |
| 事务回滚 | — | `$transaction` 内任一操作失败自动回滚，保证原子性 |

### 13.5 AI 助手异常

| 异常场景 | 处理方式 | 降级策略 |
|---------|---------|---------|
| 未配置 AI | 检查 `db.aIConfig.findFirst({ isActive: true })` | 自动降级到 `processWithKeywords`（关键词匹配） |

| API Key 无效 | 测速功能验证 | 测速失败不允许激活配置（30 分钟门控）

## 附录 B：智能补货推荐算法

### 输入数据

- 近 7 天各商品的销售总量（通过 SaleOrderItem 聚合）
- 各商品当前库存（Inventory.quantity）

### 计算公式

```
日均销量 = 近7天总销量 / 7
可售天数 = 当前库存 / 日均销量  (无销量时为 ∞)
建议采购量 = max(0, ⌈14 × 日均销量 − 当前库存⌉)
```

### 紧急程度分级

| 级别 | 条件 | 标签 | 颜色 |
|------|------|------|------|
| critical | 库存 = 0 | 缺货 | 红色 |
| urgent | 可售天数 < 3 | 紧急 | 橙色 |
| watch | 可售天数 < 7 | 关注 | 黄色 |
| sufficient | 可售天数 ≥ 7 | 充足 | 绿色 |

### 滞销商品检测

- 条件：近 7 天销量 = 0 且库存 > 50
- 输出：商品名、分类、库存量、库存金额
- 用途：提醒管理者关注积压库存，考虑促销或退货

### 核心代码（黄翔）

```typescript
// recommendation.server.ts — 智能补货推荐
export async function getRestockRecommendations() {
  const weekAgo = new Date();
  weekAgo.setDate(weekAgo.getDate() - 7);

  // 近 7 天各商品销售总量
  const salesData = await db.saleOrderItem.groupBy({
    by: ["productId"],
    _sum: { quantity: true },
    where: { saleOrder: { createdAt: { gte: weekAgo }, paymentStatus: "paid" } },
  });

  // 当前库存
  const inventories = await db.inventory.findMany({
    include: { product: { include: { category: true } } },
  });

  const recommendations = inventories.map((inv) => {
    const sale = salesData.find((s) => s.productId === inv.productId);
    const totalSold = sale?._sum.quantity || 0;
    const dailyAvg = totalSold / 7;
    const daysLeft = dailyAvg > 0 ? inv.quantity / dailyAvg : Infinity;
    const suggestedQty = Math.max(0, Math.ceil(14 * dailyAvg - inv.quantity));

    let urgency: "critical" | "urgent" | "watch" | "sufficient";
    if (inv.quantity === 0) urgency = "critical";
    else if (daysLeft < 3) urgency = "urgent";
    else if (daysLeft < 7) urgency = "watch";
    else urgency = "sufficient";

    return {
      productId: inv.productId,
      productName: inv.product.name,
      category: inv.product.category.name,
      stock: inv.quantity,
      unit: inv.product.unit,
      dailyAvg: Math.round(dailyAvg * 10) / 10,
      daysLeft: daysLeft === Infinity ? "∞" : Math.round(daysLeft * 10) / 10,
      suggestedQty,
      urgency,
    };
  });

  return recommendations
    .filter((r) => r.urgency !== "sufficient")
    .sort((a, b) => {
      const order = { critical: 0, urgent: 1, watch: 2, sufficient: 3 };
      return order[a.urgency] - order[b.urgency];
    });
}
```
